System Overview
Mentor‚ÄìMentee Monitoring System

Purpose of the System

  • Document each mentoring session in a clear and standardized format.
  • Track academic, behavioural and personal development of each mentee.
  • Support departmental and institutional reporting for quality assurance and accreditation.
  • Improve communication between mentors, mentees and parents (where applicable).

Key Features

  • Professional note-taking interface with 3D-styled, focus-driven input fields.
  • Centralized history of meetings, action items and follow-up dates.
  • Flexible filters for status, mentee, mentor, semester etc. (frontend demo here).
  • Modern visual design inspired by AI dashboards and Dora-style micro-interactions.
Mentoring compliance Outcome-based education Continuous monitoring

Typical Users

  • Mentors: Faculty members recording session notes and follow-ups.
  • Mentees: Students whose progress is being monitored.
  • Coordinators / HoD: Review consolidated reports periodically.
  • Management / IQAC: Use data for quality and accreditation reports.
